### 1-3. SEO 구조 진단
- **URL 체계 3종 혼재** (치명적):
  - 영문 슬러그: `/brokerage-fee-compare/broker-fee-comparison/` ✅
  - 한글 퍼센트인코딩: `/stock-analysis/%ec%8a%a4%ed%8e%98...` ⚠️ 공유 시 URL 파손·CTR 저하
  - 타임스탬프: `/market-catalysts/auto-20260706230328/` ❌ 키워드 0, 검색 노출 기회 상실
- SEO 플러그인 부재 → 메타 디스크립션·OG 이미지·Article 스키마·빵부스러기(BreadcrumbList) 모두 없음

- sitemap에 `<lastmod>` 없음 → 크롤 예산 낭비
- 내부링크는 있으나 토픽 클러스터(필러↔클러스터) 구조 아님

### 1-4. 강점 (계승할 것)
1. **콘텐츠 품질**: 글당 ~1,200단어, 8단 구조(무슨 일이 → 왜 중요 → 돈의 흐름 → 수혜/피해 → 반대 시나리오 → 체크포인트). 자동 발행이지만 편집 품질이 높음

3. **관점의 일관성**: "뉴스 제목보다 가격이 움직인 이유" — 차별화된 에디토리얼 포지션
4. **텍스트 중심**: 이미지 의존도가 낮아 마이그레이션 비용이 작음

## 3부. superstock.blog 설계

### 3-1. 아키텍처 결정: 정적 사이트 + 엣지 CDN (권고안)

```
[콘텐츠 파이프라인(기존 30분 자동발행)]
        │  Markdown/API로 출력 방향만 변경
        ▼
[정적 사이트 생성기 Astro] ──빌드(수 초)──▶ [Cloudflare Pages (무료)]
                                              │ 전 세계 엣지 캐시
                                              ▼
                                       TTFB 20~50ms · LCP < 1초
```

| | 현재(mystocknote) | A안: WP 유지+가속 | B안: Astro 정적(권고) |
|---|---|---|---|
| TTFB | ~980ms | ~100ms (Cloudflare APO $5/월) | **20~50ms** |
| HTML 크기 | 117KB | 117KB | **~20KB** |
| JS | jQuery 등 | 동일 | **0KB** (테마 토글 10줄만) |
| 서버비 | VPS 유지 | VPS+$5/월 | **0원** (Pages 무료) |
| 보안/관리 | 플러그인·DB 패치 | 동일 | 공격 표면 없음 |
| 자동발행 연동 | 그대로 | 그대로 | 스크립트가 md 커밋→자동 빌드 |

- **B안 권고 근거**: 콘텐츠가 텍스트 중심 + 이미 자동화 파이프라인 보유 → 파이프라인의 출력만 "WP POST"에서 "Markdown 커밋"으로 바꾸면 됨. 679개 기존 글은 WP REST API(`/wp-json/wp/v2/posts`)로 일괄 추출 가능
- 과도기: 도메인만 Cloudflare로 옮겨 APO 켜는 A안을 1주차에 적용(즉시 TTFB 10배 개선), 병행으로 B안 이관

### 3-2. 성능 예산 (Core Web Vitals p75 목표)
- LCP **≤ 1.0초** (권장선 2.5초의 40%로 설정) · INP ≤ 100ms · CLS ≤ 0.05
- 페이지 예산: HTML ≤ 30KB, CSS 1파일 ≤ 15KB, 폰트는 Pretendard 다이나믹 서브셋, 서드파티 JS는 광고 1개만(지연 로드+높이 예약으로 CLS 방어)
- 광고 슬롯은 `min-height` 고정 예약 — 금융 블로그 CLS 실패 1순위 원인 차단

### 3-3. URL·정보구조 (원본 결함 교정)
```
superstock.blog/
├── /market-catalysts/ess-demand-lgensol/     ← 영문 키워드 슬러그로 통일 (auto-타임스탬프 폐지)
├── /terms/                                    ← 용어사전 신설 (Investopedia 전략)
├── /guide/                                    ← 필러 가이드 (초보 코스 = 클러스터 허브)
├── /tools/fee-calculator/                     ← 계산기 (NerdWallet 전략, 백링크·제휴 자석)
└── /sitemap.xml (lastmod 포함)
```
- 기존 679개 글은 301 리다이렉트 맵으로 이관(검색 자산 보존)
- 토픽 클러스터: 필러(가이드) ↔ 클러스터(개별 글) 상호 링크, 앵커텍스트 다변화

### 3-4. 글 템플릿 (Investopedia 공식 + 원본 강점 결합)
1. 제목 즉답형 → 2. **3줄 요약 박스**(Key Takeaways) → 3. 목차 → 4. 본문(무슨 일→왜 지금→돈의 흐름 표→반대 시나리오 콜아웃→체크포인트) → 5. **저자·검토·최종수정일 블록** → 6. 면책조항 → 7. 관련글 3개
- 스키마: Article + BreadcrumbList + Person(저자) + WebSite. FAQPage는 AI 검색(RAG) 대응용으로만 유지
- E-E-A-T: About 페이지(운영자 경력), 정정 정책 페이지, 글마다 dateModified 노출

### 3-5. 유통·수익화 로드맵
| 단계 | 시점 | 액션 | 근거 |

| 1 | 0~1개월 | Cloudflare 이전+APO, superstock.blog 정적 사이트 오픈, 301 이관 | TTFB 10~20배 개선 |
| 2 | 1~3개월 | 네이버 채널 개설(투트랙), 용어사전 50편, 뉴스레터 시작 | PC 검색 네이버 우위 |

| 4 | 6개월+ | 유료 구독(자체 또는 네이버 프리미엄콘텐츠 입점) | 무료→유료 전환 3% 기준 설계 |

## 4부. 30분 자동 업데이트 + AI 검색(GEO) 전략 (2026-07-07 증보)

### 4-1. 자동 파이프라인 (검증 완료 — 상세는 automation/README.md)
- **무료 확정 조합**: Public GitHub 리포 + Actions 30분 크론(무제한 무료) → 수집·생성·검수 → Cloudflare Workers Static Assets 배포(요청 무제한 무료). Vercel Hobby(하루 1회+상업 금지)·Pages Git 연동(월 500빌드)은 불가 판정
- **사실성 설계**: 수치는 LLM이 아니라 템플릿 슬롯 채우기(AP통신 방식), 분석 문단만 LLM(facts JSON만 주입), 발행 전 숫자 정규식 대조로 불일치 시 발행 차단
- **데이터 경계**: 공시·뉴스 제목은 실시간 100% 사실 가능 / 시세는 전일 확정치까지가 무료·합법 / 당일 장중 시세 게시는 코스콤 계약 필요 → TradingView 위젯으로 대체
- **규제**: 무료 공개+광고수익은 유사투자자문업 신고 불요(금융위 유권해석) — 유료 구독 도입 시 재검토

### 4-2. 매일/30분 발행 SEO 시스템 (원본 about 페이지 전략의 업그레이드)
- 원본 의도 확인: "매일 장 마감 후 시장 정리 + 공개 데이터 기반" — 발행 빈도로 신선도 신호를 쌓는 전략. 계승하되 3계층으로 구조화:
  1. **30분 와이어**(자동): 공시·뉴스 기반 단신 — 신선도·롱테일 담당
  2. **일 1회 마감 브리핑**(자동+검수): 당일 재료 종합 — "오늘의 시장재료" 필러
  3. **주 1~2회 심층 분석**(수동): E-E-A-T·백링크 담당
- 구글 정책상 빈도 자체는 무관, "부가가치 없는 대량 발행"만 위반 → 모든 자동 글에 수혜/피해·반대 시나리오·체크포인트 필수 + AI 활용 고지 블록

### 4-3. 제미나이 등 AI 검색 노출 (GEO)
- **Gemini는 구글 인덱스를 읽는다** → 구글 SEO(Article 스키마·즉답 문장·수정일)가 곧 Gemini 노출 전략. ChatGPT는 Bing 인덱스 → Bing 웹마스터 등록 필수, 네이버는 서치어드바이저 등록
- 적용 완료: `robots.txt`(Google-Extended·GPTBot·PerplexityBot·ClaudeBot 허용), `llms.txt`(사이트 요약+인용 규칙), 글마다 눈에 보이는 FAQ Q&A 블록(AI가 가장 잘 인용하는 형식), 모든 수치에 출처 각주
- AI 검색은 "정의→즉답→근거" 구조의 문단을 인용한다 → 글 도입부 3줄 요약이 그대로 인용 후보
